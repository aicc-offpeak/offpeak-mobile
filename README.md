# OffPeak Mobile

## 프로젝트 개요
OffPeak Mobile은 사용자 현재 위치를 기준으로 주변의 카페·음식점 중 비교적 덜 붐비는 장소를 빠르게 탐색할 수 있도록 만든 위치기반 모바일 서비스입니다.  
기존 지도 서비스에서 원하는 정보를 빠르게 찾기 어렵다고 느낀 경험을 바탕으로, 위치 정보와 혼잡도 정보를 함께 제공하고 사용자가 여러 장소를 직관적으로 비교할 수 있는 화면을 만드는 것을 목표로 했습니다.

## 기술 스택
- Flutter
- Dart
- Kakao Map API
- REST API

## 주요 기능
- 현재 위치 기반 주변 카페·음식점 탐색
- 카카오맵 기반 장소 표시 및 마커 구성
- 장소 정보와 혼잡도 정보 제공
- 검색, 추천, 상세 확인 흐름 구성
- 상태 변화에 따른 UI 전환

## 아키텍처
- Flutter 기반 모바일 프론트엔드
- 백엔드 API 연동
- 지도 데이터 및 장소/혼잡도 정보 화면 반영

## 수행 내용
- Flutter 기반 화면 구현
- 카카오맵 API 연동
- 장소 마커 및 정보 패널 UI 구성
- 상태에 따른 화면 전환 처리
- 백엔드 API 연동 및 데이터 반영
- 검색, 추천, 상세 확인 흐름 설계

## 결과 및 성과
사용자 현재 위치를 기준으로 주변 카페·음식점 정보를 지도에서 확인하고, 혼잡도와 장소 정보를 함께 비교할 수 있는 위치기반 화면을 구현했습니다.

- 지도와 정보 패널이 연결된 탐색 흐름 구성
- 혼잡도 정보를 반영한 장소 비교 기능 구현
- 현재 위치 기반 주변 장소 탐색 기능 구현
- 검색, 추천, 상세 확인이 이어지는 사용자 흐름 구성

## 개발환경에서 빌드 및 실행 방법

1. 레포 클론
   원격 저장소를 로컬로 내려받고 프로젝트 디렉터리로 이동합니다.

git clone 레포주소
cd 레포폴더명

2. Flutter 설치 확인
   Flutter SDK가 설치되어 있는지 확인합니다.

flutter --version

3. Android 개발 환경 점검
   Android SDK, toolchain, 라이선스 상태를 확인합니다.

flutter doctor
flutter doctor --android-licenses

4. 실기기 연결 확인
   USB 디버깅이 활성화된 실기기가 인식되는지 확인합니다.

adb devices
flutter devices

5. 프로젝트 의존성 설치
   클론 직후 반드시 의존성을 설치합니다.

flutter pub get

6. 앱 실행 (실기기)
   연결된 실기기에 앱을 빌드 및 실행합니다.

flutter run

7. 코드 수정 후 반영
   실행 중에는 hot reload / restart를 사용합니다.

r
R

8. 설정·의존성 변경 후에는 앱을 종료하고 다시 실행합니다.

Ctrl + C
flutter run

9. 빌드 문제 발생 시 초기화
   캐시 문제나 빌드 오류 발생 시 사용합니다.

flutter clean
flutter pub get
flutter run

## 개발환경에서 로컬 호스팅 중인 벡엔드와 연결 방법

adb reverse tcp:(벡엔드 포트) tcp:(벡엔드 포트)
예: adb reverse tcp:8000 tcp:8000
