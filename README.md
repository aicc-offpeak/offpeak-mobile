# OffPeak Mobile

사용자 현재 위치를 기준으로 주변의 카페·음식점 중 비교적 덜 붐비는 장소를 탐색할 수 있도록 돕는 위치기반 모바일 서비스입니다.  
카카오맵 API와 혼잡도 데이터를 활용해 주변 장소를 지도에서 확인할 수 있으며, 사용자가 여러 장소를 빠르고 직관적으로 비교·탐색할 수 있도록 화면 흐름과 정보 구조를 설계했습니다.

## 프로젝트 배경
기존 지도·장소 탐색 서비스는 정보량이 많지만, 지금 바로 갈 만한 장소를 빠르게 판단하기 어려운 경우가 많았습니다.  
OffPeak는 사용자의 현재 위치를 기준으로 주변 장소와 혼잡도 정보를 함께 제공하여, 복잡한 탐색 과정 없이 더 효율적으로 선택할 수 있는 경험을 목표로 했습니다.

## 주요 기능
- 현재 위치 기반 주변 카페·음식점 탐색
- 카카오맵 기반 장소 표시 및 마커 구성
- 장소별 혼잡도 정보 및 상세 정보 제공
- 지도와 정보 패널을 연동한 탐색 흐름 구성
- 상태에 따른 UI 전환 및 사용자 흐름 최적화

## 담당 역할
본 프로젝트에서 프론트엔드 구현을 주도했습니다.

- Flutter 기반 모바일 화면 구현
- 카카오맵 API 연동
- 백엔드 API 연동 및 데이터 화면 반영
- UI/UX 및 사용자 흐름 설계
- 지도/정보 패널 전환, 혼잡도 표현 방식, 검색·추천 흐름 조정

## 구현 과정에서 중점적으로 고민한 점
이 프로젝트에서는 단순히 지도에 장소를 표시하는 것보다,  
사용자가 **지금 필요한 정보를 빠르게 이해하고 바로 비교·선택할 수 있는 흐름**을 만드는 데 집중했습니다.

특히 다음과 같은 부분을 반복적으로 조정했습니다.

- 초기 진입 시 지도 카메라와 마커 노출 방식
- 지도와 정보 패널 간 전환 흐름
- 혼잡도 정보의 시각적 표현 방식
- 사용자가 덜 헷갈리도록 만드는 검색·추천·상세 확인 흐름
- 상태 변화에 따른 UI 반응과 정보 노출 방식

이를 통해 위치기반 서비스에서는 정보량 자체보다,  
**정보의 순서와 표현 방식, 화면 흐름을 어떻게 설계하느냐가 사용자 경험에 더 큰 영향을 준다**는 점을 확인했습니다.

## 개발환경에서 빌드 및 실행 방법

1. 레포 클론
   원격 저장소를 로컬로 내려받고 프로젝트 디렉터리로 이동합니다.

git clone 레포주소
cd 레포폴더명

2. Flutter 설치 확인
   Flutter SDK가 설치되어 있는지 확인합니다.

flutter --version

3. Android 개발 환경 점검
   Android SDK, toolchain, 라이선스 상태를 확인합니다.

flutter doctor
flutter doctor --android-licenses

4. 실기기 연결 확인
   USB 디버깅이 활성화된 실기기가 인식되는지 확인합니다.

adb devices
flutter devices

5. 프로젝트 의존성 설치
   클론 직후 반드시 의존성을 설치합니다.

flutter pub get

6. 앱 실행 (실기기)
   연결된 실기기에 앱을 빌드 및 실행합니다.

flutter run

7. 코드 수정 후 반영
   실행 중에는 hot reload / restart를 사용합니다.

r
R

8. 설정·의존성 변경 후에는 앱을 종료하고 다시 실행합니다.

Ctrl + C
flutter run

9. 빌드 문제 발생 시 초기화
   캐시 문제나 빌드 오류 발생 시 사용합니다.

flutter clean
flutter pub get
flutter run
